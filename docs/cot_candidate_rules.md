# CoT 候选题筛选规则（第一阶段）

> 更新时间：2026-05-13
> 适用范围：`data/processed/cmb_clean/cmb_train_vector_dedup_bucket.jsonl`
> 目标：从 clean + dedup pool 中筛出“更适合显式推理 CoT”的题，而不是单纯筛“最长”或“最难”的题。

---

## 1. 设计目标

第一阶段规则筛选的目标不是给题目打一个绝对真实的难度标签，而是构建一个可解释、可复现、低成本的 `CoT candidate score`，优先保留：

- 多选题
- 病例信息较丰富的题
- 选项之间较难区分的题
- 更可能需要比较、排除、整合信息的题

同时降低以下题目的优先级：

- 纯定义/概念识别题
- 题干推理证据不足、主要依赖直接 recall 的题

---

## 2. 总体公式

```text
cot_candidate_score
= 2.0 * is_multi_choice
+ case_info_score
+ option_confusion_score
- definition_penalty
- low_reasoning_penalty
```

解释：

- `is_multi_choice`：多选题加分
- `case_info_score`：题干覆盖的病例信息类别越多，加分越高
- `option_confusion_score`：选项越相似、越难区分，加分越高
- `definition_penalty`：纯定义/概念题降权
- `low_reasoning_penalty`：题干缺少推理证据时降权

---

## 3. 各项定义

### 3.1 `is_multi_choice`

规则：

- `question_type` 含 `多项选择题` 或答案长度大于 1：记为 `1`
- 否则记为 `0`

计分：

- 多选题：`+2.0`
- 单选题：`+0.0`

设计理由：

- 多选题天然更适合 CoT，因为通常需要逐项判断、排除和组合答案。

### 3.2 `case_info_score`

核心思想：

- 不用“是否出现患者”这种单点词判断病例题。
- 改用“题干覆盖了多少类病例信息”来估计信息密度。

建议分为 4 类：

1. 人群信息
   - 例：`男`、`女`、`患者`、`患儿`、`孕妇`、`老年`、`儿童`
2. 症状体征
   - 例：`发热`、`腹痛`、`呕吐`、`咳嗽`、`胸痛`、`浮肿`、`头痛`
3. 检查检验
   - 例：`查体`、`血压`、`脉搏`、`体温`、`X线`、`CT`、`MRI`、`实验室`
4. 时间/过程信息
   - 例：`2小时`、`10天`、`反复`、`突发`、`进行性`、`半年`

计分建议：

- 覆盖 0 类：`0.0`
- 覆盖 1 类：`0.5`
- 覆盖 2 类：`1.0`
- 覆盖 3 类及以上：`1.5`

设计理由：

- 真正值得生成 CoT 的题，往往不是“题干长”，而是同时包含人群、症状、检查、病程等多类信息。

### 3.3 `option_confusion_score`

核心思想：

- 选项越像，区分越难，越适合显式推理。
- 第一阶段不引入语义模型，先用词面层面的混淆度做代理特征。

建议由两部分组成：

#### A. 长度均衡分

思路：

- 如果各选项长度差不多，通常比“一长四短”更难。

实现建议：

- 计算每个选项的字符长度
- 计算变异系数：`std(lengths) / mean(lengths)`

计分建议：

- 变异系数低：`+0.8`
- 变异系数中等：`+0.4`
- 变异系数高：`+0.0`

#### B. 选项两两二元匹配分

思路：

- 用字符二元组（bigram）比较选项之间的词面相似度。
- 选项共享的二元片段越多，越可能属于同一类候选，混淆度越高。

实现建议：

- 将每个选项转成字符 bigram 集合
- 计算任意两个选项的 Jaccard 相似度
- 对所有选项对求平均相似度

计分建议：

- 平均相似度高：`+0.8`
- 平均相似度中等：`+0.4`
- 平均相似度低：`+0.0`

#### `option_confusion_score` 合成

```text
option_confusion_score
= length_balance_score
+ pairwise_bigram_similarity_score
```

总范围建议：

- `0.0 ~ 1.6`

设计理由：

- 第一版先用可解释的词面相似度，不依赖 teacher 或 embedding。
- 后续如果需要升级，可以换成更细的医学术语重叠、分词相似度或 embedding 相似度。

### 3.4 `definition_penalty`

核心思想：

- 纯定义/概念题往往更像知识点 recall，不一定适合长 CoT。
- 但不能把所有“特点/特征”题都重罚，因为其中一部分选项仍然很有混淆度。

分成两档：

#### 强定义模板

匹配词：

- `什么是`
- `什么指`
- `概念`
- `定义`
- `含义`

建议惩罚：

- `-1.0`

#### 弱定义模板

匹配词：

- `特点`
- `特征`

建议惩罚：

- `-0.3 ~ -0.5`

设计理由：

- 数据中确实存在大量 `概念/定义/含义` 型题干。
- `特点/特征` 题需要保守处理，只做轻降权。

### 3.5 `low_reasoning_penalty`

核心思想：

- 这里不是判“简单题”，而是判“题干中缺少需要展开显式推理的证据”。
- 不重复使用 `is_multi_choice` 和 `option_confusion_score`，避免双重计分。

只看题干本身是否符合多个低推理证据条件。

建议条件：

1. 题干长度较短，例如 `< 20`
2. `case_info_score = 0`
3. 匹配 recall 模板：
   - `最常见`
   - `首选`
   - `适应证`
   - `禁忌证`
   - `作用`
   - `机制`
   - `第一部`
4. 不匹配病例信息模板

计分建议：

- 满足 0-1 项：`0.0`
- 满足 2 项：`0.5`
- 满足 3 项及以上：`1.0`

设计理由：

- 有些题虽然出现了 `最可能`、`首选` 等词，但如果题干本身是病例题，就不应直接降权。
- 所以 `low_reasoning_penalty` 必须与 `case_info_score` 联合使用。

## 4. 基于真实数据观察到的题干模式

以下模式来自对 `cmb_train_clean.jsonl` 的真实抽样观察。

### 4.1 定义/概念题常见模式

- `下列关于胶囊概念的正确叙述是`
- `“血为气母”的含义是`
- `小气道的概念是`
- `什么是声影`
- `什么是动物细胞进行生物氧化和能量转换的主要场所`

结论：

- `什么是 / 什么指 / 概念 / 定义 / 含义` 可以作为强定义模板。

### 4.2 弱定义题常见模式

- `心源性哮喘的特点是`
- `猩红热的皮疹特点是`
- `支气管鳞状上皮细胞癌的特点是`
- `细胞因子所不具备的作用特点是`

结论：

- `特点 / 特征` 题数量很多，不能全部当低价值题，宜轻罚。

### 4.3 低推理 recall 题常见模式

- `治疗艾滋病患者隐球菌性脑膜炎的首选药是`
- `引起原发性甲状旁腺功能亢进最常见的疾病是`
- `婴幼儿化脓性脑膜炎最常见的病原菌是`
- `风湿热患者以发热或关节炎为主症时,首选治疗药物是`

结论：

- `最常见 / 首选 / 适应证 / 禁忌证 / 作用 / 机制 / 第一部` 可以作为 recall 模板。
- 但必须和 `case_info_score` 联合看，避免误伤病例题。

### 4.4 病例题常见模式

- `患者,60岁,男性,突发头痛、呕吐、视物旋转伴行走不稳2小时...`
- `男性,78岁,排尿困难12年,近1年常出现排尿中断现象...`
- `患者女,42岁,鼻塞、流脓涕10天,视力下降4天...`
- `患者,女,29岁。右下智齿反复肿痛...X线片示...`

结论：

- 病例题往往同时覆盖人群、症状、检查、病程等多个类别，适合作为 CoT 候选。

## 5. 误判风险与处理原则

### 风险 1：短题不一定是纯记忆题

例子：

- `HIV患者最常感染的是下列哪种肺炎`

结论：

- 短题只是弱信号，不能单独作为降权依据。

### 风险 2：`最可能`、`首选` 等词不能直接视为低推理

例子：

- `患儿男,8岁...最可能的诊断是`

结论：

- 如果存在病例信息，`最可能` 类题往往反而适合 CoT。

### 风险 3：`特点 / 特征` 题不一定低价值

结论：

- 只轻降权，不做强惩罚。

### 风险 4：定义题与难题不是一回事

结论：

- 第一阶段筛的是“是否适合 CoT”，不是“题目是否绝对困难”。

---

## 6. 推荐执行流程

1. 从 `cmb_train_vector_dedup_bucket.jsonl` 读取候选池
2. 为每道题计算：
   - `is_multi_choice`
   - `case_info_score`
   - `option_confusion_score`
   - `definition_penalty`
   - `low_reasoning_penalty`
3. 计算 `cot_candidate_score`
4. 按分数排序
5. 先抽 top 一批作为 CoT 候选池
6. 第二阶段再用 teacher 模型做精筛

---

## 7. 面试表述

可以这样讲：

> 我没有直接把所有医学题都拿去生成 CoT，而是先做了一个可解释的第一阶段筛选。这个筛选不是简单按题长或关键词，而是构建了一个 CoT candidate score，综合考虑题型、多源病例信息、选项混淆度，以及纯定义/低推理题降权。这样筛出来的不是“最长的题”，而是更适合显式推理监督的题。后续我再用 teacher 模型做第二阶段精筛，进一步提升 CoT 子集质量。
