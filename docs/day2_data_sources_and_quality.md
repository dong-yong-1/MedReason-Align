## 3. 数据源清单

### A. 主候选池（优先）

#### 1) FreedomIntelligence/HuatuoGPT-sft-data-v1

- 类型：中文医疗 SFT 指令数据
- 定位：候选池主力之一
- 用途：
  - 提供大量中文医疗样本
  - 从中筛“像病例推理”的数据
  - 保留少量做通用医疗问答补充
- 优点：
  - 量大
  - 中文为主
  - 医疗场景覆盖较广
- 风险：
  - 风格偏问答，不天然是病例推理
  - 需要重写成目标 schema


#### 2) shibing624/medical

- 类型：中文医疗综合数据集
- 定位：大候选池
- 用途：
  - 提供广覆盖医疗问答/指令样本
  - 做目标分布相似度筛选
  - 补充训练覆盖面
- 优点：
  - 量足
  - 适合作为“原料池”
- 风险：
  - 数据风格不统一
  - 质量波动较大
  - 需要严格过滤

### B. 病例/考试风格数据（强相关）

#### 3) FreedomIntelligence/CMB

- 类型：中文医学考试与临床评测数据
- 定位：任务骨架来源
- 用途：
  - 提供更像病例分析题的输入风格
  - 用作目标分布 seed 的重要来源
  - 部分公开集可用作外部 benchmark
- 优点：
  - 更接近“病例推理”
  - 更适合你的主任务
- 风险：
  - 训练集与评测集必须严格隔离
  - 不要把最终测试题风格直接泄漏进训练构造

#### 4) MedQA-USMLE-4-options-hf（可选）

- 类型：英文医学考试题
- 定位：补充来源
- 用途：
  - 少量补充高质量医学 reasoning 样本
  - 做跨语言泛化验证
- 优点：
  - 题目规范
  - 有一定推理价值
- 风险：
  - 英文风格与中文目标分布存在偏差
  - 第一版不要占比过大

### C. 自建数据（高价值）

#### 5) 手工 seed set

- 类型：你自己整理的高质量标准样本
- 定位：最关键的数据资产
- 用途：
  - 定义目标任务分布
  - 做相似度匹配的“锚点”
  - 用于 teacher 改写模板打样
- 建议规模：
  - 第一版 50~100 条
  - 后续扩到 200~500 条
- 优点：
  - 最能体现你的任务理解
  - 是整个项目的“方向盘”
- 风险：
  - 前期耗时
  - 需要你自己认真定义 schema

## 5. 质量标准：什么样的数据能留

### 5.1 样本级质量标准

每条候选样本至少满足：

- 与医学相关
- 不是纯闲聊
- 能抽取出明确病例信息或问题背景
- 不含明显乱码或缺字段
- 不依赖图片、表格、外部附件才能理解
- 文本长度合理，不要过短

### 5.2 任务相关性标准

如果一条样本想进入病例推理训练集，最好满足下列至少 2~3 条：

- 有症状描述
- 有病程信息
- 有既往史 / 用药史 / 检查信息
- 可以归纳出诊断方向
- 可以延展出鉴别诊断
- 可以给出下一步建议

### 5.3 安全性标准

剔除或降权以下数据：

- 明显宣称“线上直接确诊”
- 明显提供高风险处方建议
- 缺少风险边界且语气过度肯定
- 内容与循证医学常识明显冲突
- 有明显幻觉或编造信息

### 5.4 结构化可改写标准

优先保留：

- 能被改写成固定 schema 的样本
- 问题和答案之间因果关系清晰
- 适合生成主诊断/依据/鉴别/建议/风险字段

不优先保留：

- 情绪安慰型对话
- 与病例推理弱相关的泛科普
- 纯药品说明书式内容
- 过于碎片化、无法构成病例的单句问答

## 6. 质量过滤规则（第一版）

### 硬过滤

直接删除：

- 空样本
- 字段缺失严重
- 乱码
- 重复样本
- 极短样本
- 非医疗样本
- 依赖外部上下文才能理解的样本

### 软过滤

保留但降权：

- 只有问答，没有病例背景
- 只有结论，没有依据
- 只有一般建议，没有鉴别信息
- 风险提示不明显的样本

### 优先保留

- 有典型病例叙述结构
- 有多项症状和病史信息
- 容易构造成临床推理任务
- 能支持结构化输出

## 7. 数据风险清单

你今天必须明确写下以下风险：

### 风险 1：数据泄漏

- 不要把最终 benchmark 测试集直接用于筛选训练样本
- `target-dev-distribution set` 和 `final holdout` 必须分开

### 风险 2：任务漂移

- 医疗问答数据太多会把任务带偏成“泛聊天”
- 第一版必须控制通用问答占比

### 风险 3：teacher 幻觉

- 后续 teacher 改写会编造“未给出的病例细节”
- 因此需要人工抽样检查

### 风险 4：类别失衡

- 如果全是呼吸、消化病例，模型会偏科
- 需要在 seed 和 holdout 里做科室覆盖

### 风险 5：输出过度自信

- 医疗任务最怕模型语气太绝对
- 所以后续偏好数据必须约束保守表达

## 11. Day 2 定稿版结论

这一节不是“可选建议”，而是第一版项目正式采用的方案。

### 11.1 最终采用的数据源

#### A. 训练候选池

第一版正式采用以下 3 个来源：

1. `FreedomIntelligence/HuatuoGPT-sft-data-v1`
2. `shibing624/medical`
3. `FreedomIntelligence/CMB` 的训练部分

说明：

- `HuatuoGPT-sft-data-v1` 负责提供大量中文医疗指令样本
- `shibing624/medical` 负责提供大规模医疗候选池
- `CMB train` 负责提供更接近病例推理风格的任务骨架

#### B. 辅助候选池

第一版可选少量引入：

4. `GBaker/MedQA-USMLE-4-options-hf`

说明：

- 仅做少量补充
- 占比控制在很低水平
- 只用于增强医学 reasoning 风格，不作为中文主力来源

#### C. 自建数据资产

第一版必须自己做两份数据：

1. `target-dev-distribution set`
2. `final holdout eval set`

说明：

- 这两份数据都不能缺
- 如果没有这两份数据，你这个项目就没有“自己的任务定义”和“自己的评测闭环”

### 11.2 第一版数据角色划分

| 数据源 | 最终角色 | 第一版是否必须 |
|---|---|---|
| HuatuoGPT-sft-data-v1 | 大规模候选池 | 是 |
| shibing624/medical | 大规模候选池 | 是 |
| CMB train | 病例推理骨架来源 | 是 |
| MedQA | 少量辅助 | 否 |
| target-dev-distribution set | 目标分布定义 | 是 |
| final holdout eval set | 最终盲评 | 是 |

### 11.3 第一版不采用的内容

为了避免任务跑偏，以下内容第一版不作为重点：

- 纯情绪安慰型医患对话
- 泛医疗科普长文
- 纯药品说明书式文本
- 与病例推理无关的工具调用样本
- 只适合预训练、不适合病例分析的纯文档语料

### 11.4 第一版训练集构成定稿

第一版 SFT 数据目标构成如下：

- 65%：病例推理样本
- 25%：高相关医疗问答样本
- 10%：安全边界 / 风险提示样本

第一版 DPO 数据目标构成如下：

- 主体围绕病例推理回答偏好
- 明确比较：
  - 完整 vs 不完整
  - 保守 vs 过度自信
  - 有风险提示 vs 无风险提示
  - 有鉴别诊断 vs 无鉴别诊断

### 11.5 第一版质量标准定稿

#### 可以进入候选池

满足以下至少一条：

- 包含病例描述
- 包含症状、病程、既往史中的任意两项
- 是医学题目，且题干具有病例分析特征
- 能够被改写为“诊断方向 + 依据 + 鉴别 + 建议 + 风险”格式

#### 可以进入训练集

满足以下至少四条：

- 明确是医疗场景
- 有足够的病例信息
- 可以推导出主诊断方向
- 可以写出至少 2 条诊断依据
- 可以写出至少 1 条鉴别诊断
- 可以写出下一步建议
- 可以写出风险提示

#### 直接删除

满足任一条就删：

- 非医疗内容
- 乱码或字段缺失严重
- 重复样本
- 过短样本
- 明显依赖图片/表格/附件
- 与病例推理明显无关
- 明显包含错误医学信息

#### 保留但降权

- 普通医疗问答
- 依据不完整的样本
- 缺少风险提示的样本
- 结构松散但可重写的样本

### 11.6 数据泄漏规则定稿

以下规则正式生效：

- `final holdout eval set` 绝不参与训练
- `final holdout eval set` 绝不参与相似度筛选
- `final holdout eval set` 绝不参与 teacher 改写
- `target-dev-distribution set` 不作为最终测试集
- `CMB` 若用于评测，则训练和测试部分必须严格分离
