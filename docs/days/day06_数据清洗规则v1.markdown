# Day 6：数据清洗规则 v1

目标：
为 `MedReason-Align` 项目建立一套稳定、可复现、可解释的数据治理标准，用于后续数据筛选、脚本实现、人工复核和实验记录。

项目主任务：
`结构化病例诊断推理`

这份文档记录的是“规则标准”，不是具体脚本实现。
原因很简单：

- 规则定义“什么该留、什么该删、什么要复核”
- 流程定义“这些规则按什么顺序执行”

第一版先把标准定下来，后面再写清洗 pipeline 和脚本实现。

---

## 1. 适用范围

这份规则适用于以下数据：

- 原始候选池样本
- `train_candidate`
- `target_dev` 构造阶段的候选样本
- `final_holdout` 构造阶段的候选样本

其中：

- 训练候选样本重点执行去重、去泄漏、去低质
- `target_dev` 和 `final_holdout` 更强调质量和隔离，不追求大规模

---

## 2. 样本最终状态

每条样本经过规则处理后，必须落入以下 5 类状态之一：

- `accepted`
- `rejected_duplicate`
- `rejected_leakage`
- `rejected_low_quality`
- `needs_manual_review`

后续统计和实验日志都以这 5 类状态为准。

---

## 3. 规则分组

第一版规则分为 5 组：

- `R-NORM-*`：标准化前处理
- `R-DUP-*`：去重规则
- `R-LEAK-*`：去泄漏规则
- `R-QUAL-*`：低质过滤规则
- `R-REVIEW-*`：人工复核规则

---

## 4. R-NORM：标准化规则

这组规则不是过滤，而是所有后续判断的统一前处理。

### R-NORM-001 文本标准化

- 适用对象：所有候选样本
- 判断条件：进入清洗流程前统一处理
- 处理动作：生成 `case_text_norm` 和 `schema_target_norm`
- 备注：第一版至少包含以下操作
  - 全角半角统一
  - 去多余空格
  - 去重复换行
  - 标点统一

### R-NORM-002 联合文本生成

- 适用对象：所有候选样本
- 判断条件：完成 `R-NORM-001` 后执行
- 处理动作：生成 `joint_text_norm = case_text_norm + schema_target_norm`
- 备注：一级精确去重以 `joint_text_norm` 为准

---

## 5. R-DUP：去重规则

去重规则分为三级：

- 一级：联合完全重复
- 二级：embedding 近重复
- 三级：模板重复

### R-DUP-001 联合完全重复去重

- 适用对象：所有候选样本
- 判断条件：`joint_text_norm` 完全一致
- 处理动作：保留质量更高、信息更完整的一条，其余标记为 `rejected_duplicate`
- 备注：这是一级去重主规则，必须执行

### R-DUP-002 embedding 近重复去重

- 适用对象：通过一级去重后的样本
- 判断条件：样本之间的 embedding 相似度高于阈值
- 处理动作：
  - 若相似度 `>= T_dup_high`，直接标记为 `rejected_duplicate`
  - 若相似度在 `T_dup_mid ~ T_dup_high`，进入 `needs_manual_review`
- 备注：这是二级去重主规则

第一版阈值草案：

- `T_dup_high = 0.90`
- `T_dup_mid = 0.80`

### R-DUP-003 模板重复候选识别

- 适用对象：通过一级、二级去重后的样本
- 判断条件：满足以下多项特征之一
  - `primary_diagnosis` 相同
  - `case_text` 结构高度一致
  - 只改年龄、病程、少量症状词
  - `diagnostic_basis` 大量复用
- 处理动作：进入 `needs_manual_review`
- 备注：这是三级去重的触发规则，第一版不直接自动删除

### R-DUP-004 模板簇限流

- 适用对象：人工确认后的模板重复样本簇
- 判断条件：同一模板簇内部样本高度同质
- 处理动作：每个模板簇只保留少量代表样本，其余标记为 `rejected_duplicate`
- 备注：优先保留
  - 信息更完整的
  - 风险更清晰的
  - 更贴近目标输入风格的

### 三级模板重复当前说明

目前“模板重复去重”先作为复核型规则，不强求第一版完全自动化实现。

原因：

- 模板重复往往不是文本完全相同
- 它更多是“病例结构高度相似”
- 直接自动删除，容易误删有价值样本

所以第一版策略是：

- 先识别候选模板簇
- 再人工复核
- 最后做模板簇限流

---

## 6. R-LEAK：去泄漏规则

这组规则用于保证评测可信度。

### R-LEAK-001 公共测试集硬隔离

- 适用对象：所有训练候选样本
- 判断条件：样本来自公共 benchmark 的 `test split`
- 处理动作：直接标记为 `rejected_leakage`
- 备注：不得进入训练集，不得用于 teacher 改写扩增

### R-LEAK-002 final_holdout 精确重复拦截

- 适用对象：训练候选样本
- 判断条件：与 `final_holdout` 中任一样本的 `joint_text_norm` 完全一致
- 处理动作：直接标记为 `rejected_leakage`
- 备注：这是最严格的泄漏拦截规则

### R-LEAK-003 final_holdout 近泄漏拦截

- 适用对象：训练候选样本
- 判断条件：与 `final_holdout` 样本 embedding 相似度高于阈值
- 处理动作：
  - 若相似度 `>= T_leak_high`，直接标记为 `rejected_leakage`
  - 若相似度在 `T_leak_mid ~ T_leak_high`，进入 `needs_manual_review`
- 备注：泄漏判断优先看病例信息是否实质相同，不只看表面措辞

第一版阈值草案：

- `T_leak_high = 0.92`
- `T_leak_mid = 0.85`

### R-LEAK-004 target_dev 核心样本隔离

- 适用对象：训练候选样本
- 判断条件：与已锁定的 `target_dev` 样本精确重复或高相似
- 处理动作：
  - 精确重复直接 `rejected_leakage`
  - 高相似进入 `needs_manual_review`
- 备注：`target_dev` 可以用于开发校准，但不应反向污染训练集

### R-LEAK-005 eval 改写样本禁止入训

- 适用对象：所有人工或模型改写样本
- 判断条件：样本直接或间接由 `final_holdout` 或锁定 `target_dev` 改写而来
- 处理动作：直接标记为 `rejected_leakage`
- 备注：改写、扩写、摘要化、同义翻写都视为泄漏风险

---

## 7. R-QUAL：低质过滤规则

这组规则是质量门，重点防止模型学到坏信号。

### 7.1 结构低质

#### R-QUAL-001 无法映射到 schema v1

- 适用对象：所有候选样本
- 判断条件：无法稳定构造 `schema_target`
- 处理动作：标记为 `rejected_low_quality`
- 备注：若连 `primary_diagnosis` 和 `diagnostic_basis` 都构不出来，直接剔除

#### R-QUAL-002 结构信息不足

- 适用对象：所有候选样本
- 判断条件：病例背景极弱，无法支持病例推理
- 处理动作：标记为 `rejected_low_quality`
- 备注：如“感冒怎么办”这类泛提问不进入核心病例集

### 7.2 医学内容低质

#### R-QUAL-003 医学常识明显错误

- 适用对象：所有候选样本
- 判断条件：结论与病例信息明显不符，或存在明确医学错误
- 处理动作：标记为 `rejected_low_quality`
- 备注：这类样本不能作为正向训练信号

#### R-QUAL-004 高风险病例缺少风险提示

- 适用对象：高风险候选样本
- 判断条件：病例具有明显红旗特征，但 `risk_flags` 为空或明显错误
- 处理动作：标记为 `rejected_low_quality` 或进入强制重写队列
- 备注：第一版核心训练集里建议直接剔除

### 7.3 推理低质

#### R-QUAL-005 推理依据不足

- 适用对象：所有候选样本
- 判断条件：`diagnostic_basis` 少于 2 条，或内容明显空泛
- 处理动作：标记为 `rejected_low_quality` 或降级为弱监督候选
- 备注：例如“根据患者情况考虑”不算有效依据

#### R-QUAL-006 鉴别诊断无区分价值

- 适用对象：所有候选样本
- 判断条件：`differential_diagnoses` 与主诊断高度重复，或乱列无关病种
- 处理动作：标记为 `rejected_low_quality` 或进入重写队列
- 备注：鉴别诊断不能只是凑数

#### R-QUAL-007 建议不可用或明显不安全

- 适用对象：所有候选样本
- 判断条件：
  - `recommended_actions` 只有空泛建议
  - 缺少必要检查或就医动作
  - 存在明显不安全建议
- 处理动作：标记为 `rejected_low_quality`
- 备注：如高风险胸痛只写“注意休息”必须剔除

### 7.4 表达低质

#### R-QUAL-008 表达噪声严重

- 适用对象：所有候选样本
- 判断条件：
  - 乱码严重
  - 中英混杂影响理解
  - 机器翻译腔明显且难以标准化
- 处理动作：标记为 `rejected_low_quality`
- 备注：第一版优先保留可稳定映射的中文病例文本

### 7.5 安全边界低质

#### R-QUAL-009 过度自信表达

- 适用对象：所有候选样本
- 判断条件：在信息不足时仍使用绝对化措辞
- 处理动作：标记为 `rejected_low_quality` 或进入重写队列
- 备注：如“肯定就是”“无需检查即可确诊”

### 7.6 分布低质

#### R-QUAL-010 非目标分布样本降权

- 适用对象：所有候选样本
- 判断条件：虽为医疗内容，但与病例推理任务弱相关
- 处理动作：不进入核心训练集，可留在通用补充池
- 备注：例如泛科普、情绪安慰、药品百科

---

## 8. R-REVIEW：人工复核规则

边界样本不自动删除，进入人工复核。

### R-REVIEW-001 中相似度近重复复核

- 适用对象：候选样本
- 判断条件：embedding 相似度位于人工复核区间
- 处理动作：标记为 `needs_manual_review`
- 备注：防止误删真正有价值的相似病例

### R-REVIEW-002 模板簇复核

- 适用对象：模板重复候选样本
- 判断条件：被 `R-DUP-003` 命中
- 处理动作：标记为 `needs_manual_review`
- 备注：由人工判断是否属于同一模板簇

### R-REVIEW-003 边界泄漏复核

- 适用对象：训练候选样本
- 判断条件：与 `final_holdout` 或锁定 `target_dev` 高相似但未达到硬拦截阈值
- 处理动作：标记为 `needs_manual_review`
- 备注：优先检查病例槽位是否本质相同

### R-REVIEW-004 高风险标签复核

- 适用对象：高风险样本
- 判断条件：`risk_level`、`risk_flags`、`recommended_actions` 三者明显不一致
- 处理动作：标记为 `needs_manual_review`
- 备注：这类问题会直接影响项目安全性结论

### R-REVIEW-005 质量边界样本复核

- 适用对象：所有候选样本
- 判断条件：样本基本可用，但在依据、鉴别或建议上存在边界问题
- 处理动作：标记为 `needs_manual_review`
- 备注：复核后决定“重写保留”还是“直接剔除”

## 9. 第一版执行建议

这部分不是规则本身，而是建议的执行顺序：

```text
raw data
-> R-NORM
-> R-DUP-001
-> R-QUAL hard filters
-> R-DUP-002
-> R-DUP-003
-> R-LEAK
-> R-REVIEW
-> final status
```

对应解释：

1. 先标准化
2. 先做联合完全重复去重
3. 再剔除明显低质样本
4. 再做 embedding 近重复
5. 再识别模板重复候选
6. 再做泄漏检查
7. 最后把边界样本放入人工复核队列

---

## 10. 记录建议

后续每条样本建议保存如下处理结果：

```json
{
  "sample_id": "cand_001",
  "final_status": "needs_manual_review",
  "hit_rules": [
    "R-DUP-003",
    "R-REVIEW-002"
  ]
}
```

或者：

```json
{
  "sample_id": "cand_002",
  "final_status": "rejected_low_quality",
  "hit_rules": [
    "R-QUAL-004",
    "R-QUAL-007"
  ]
}
```

这样后面可以做非常清楚的统计：

- 删除了多少重复样本
- 拦截了多少泄漏样本
- 剔除了多少低质样本
- 有多少样本进入人工复核

---

## 11. 第一版结论

今天正式定下：

- 去重采用三级策略：联合完全重复、embedding 近重复、模板重复复核
- 泄漏检查重点保护 `final_holdout` 和锁定 `target_dev`
- 低质过滤围绕结构、医学内容、推理质量、表达质量、安全边界展开
- 模板重复第一版先做“识别 + 人工复核”，不强行全自动

一句话总结：

> 数据清洗规则 v1 的目标不是“尽量多删数据”，而是保证训练信号干净、评测隔离可信、项目结论可复现。
